<a name="code"><h2>Код и подсветка синтаксиса</h2></a>

Блоки кода являются частью функций Markdown, но не подсветка синтаксиса. Однако многие обработчики, например Github или *Markdown Here*, поддерживают подсветку синтаксиса. Список поддерживаемых языков и способ их указания может различаться. *Markdown Here* поддерживает десятки языков (и не-языков, например синтаксис diff и заголовки HTTP); полный список и способ указания языков см. на странице [highlight.js demo-странице](http://softwaremaniacs.org/media/soft/highlight/test.html).

<a name="tables"><h2>Таблицы</h2></a>

Таблицы не являются частью Markdown, но многие обработчики, например *Markdown Here* и Github, поддерживают их. Они позволяют легко добавить таблицы в электронное письмо -- в других случаях для этого нужно копировать их из другого приложения.

```no-highlight
Вертикальные линии обозначают столбцы.

| Таблицы       | Это                | Круто |
| ------------- |:------------------:| -----:|
| столбец 3     | выровнен вправо    | $1600 |
| столбец 2     | выровнен по центру |   $12 |
| зебра-строки  | прикольные         |    $1 |

Внешние вертикальные линии (|) не обязательны, и они нужны только чтобы сам код Markdown выглядел красиво. Тот же код можно записать так:

Markdown | не такой | красивый
--- | --- | ---
*Но выводится* | `так же` | **клево**
1 | 2 | 3
```

<a name="lines"><h2>Новая строка</h2></a>

Для понимания работы разрыва строка автор главным образом рекомендует экспериментировать и пробовать -- нажмите &lt;Enter&gt; один раз (т.е. перейдите на новую строку), потом нажмите дважды (т.е. вставьте две новые строки) и посмотрите что приозошло. Вы сразу поймете что вам нужно. В расширении [Markdown Here](https://github.com/adam-p/markdown-here) для браузеров есть удобная функция "Markdown Toggle", которая поможет в этом.

*Примечание переводчика:*  
*Для переноса на новую строку в конце предыдущей строки необходимо добавить* **два пробела**. *Без этого большинство парсеров Markdown не выполняют переход на новую строку.*

Попробуйте ввести следующее:

```
Это начальная строка

Эта строка отделена от предыдущей двумя новыми строками и станет *отдельным абзацем*.

Это тоже отдельный абзац, но...⋅⋅
Эта строка отделена одной новой строкой, поэтому она находится в *том же абзаце*.
```

Это начальная строка

Эта строка отделена от предыдущей двумя новыми строками и станет *отдельным абзацем*.

Это тоже отдельный абзац, но...`[здесь два пробела]`  
Эта строка отделена одной новой строкой, поэтому она находится в *том же абзаце*.  

(Примечание: В [Markdown Here](https://github.com/adam-p/markdown-here) разрывы строк ведут себя так же, как в [GFM](https://help.github.com/articles/github-flavored-markdown), поэтому не нужно вставлять по две новые строки.)

# Справочник по командам гита

## 1. Основные команды гита

### 1.1. git init

Команда `git init` инициализирует новый пустой репозиторий Git.

```sh
git init
```

### 1.2. git add

Команда `git add` добавляет файлы в индекс Git.

```sh
git add filename.ext
```

### 1.3. git commit

Команда `git commit` создает новый коммит с описанием изменений.

```sh
git commit -m "Описание изменений"
```

### 1.4. git push

Команда `git push` отправляет изменения на удаленный сервер.

```sh
git push origin main-branch
```

### 1.5. git pull

Команда `git pull` получает изменения с удаленного сервера.

```sh
git pull origin main-branch
```

### 1.6. git clone

Команда `git clone` создает локальную копию удаленного репозитория.

```sh
git clone https://github.com/username/repo.git
```

### 1.7. git branch

Команда `git branch` создает новую ветку.

```sh
git branch new-branch
```

### 1.8. git checkout

Команда `git checkout` переключает на определенную ветку.

```sh
git checkout main-branch
```

### 1.9. git status

Команда `git status` показывает текущее состояние репозитория.

```sh
git status
```

### 1.10. git log

Команда `git log` показывает историю коммитов.

```sh
git log
```

## 2. Дополнительные команды гита

### 2.1. git tag

Команда `git tag` создает тег для определенного коммита.

```sh
git tag v1.0
```

### 2.2. git rebase

Команда `git rebase` переносит изменения из одной ветки в другую.

```sh
git rebase main-branch
```

### 2.3. git reset

Команда `git reset` отменяет изменения в индексе Git.

```sh
git reset --hard HEAD~1
```

### 2.4. git revert

Команда `git revert` создает новый коммит, который отменяет изменения предыдущего коммита.

```sh
git revert HEAD
```

### 2.5. git stash

Команда `git stash` временно сохраняет изменения в индексе Git.

```sh
git stash
```

### 2.6. git diff

Команда `git diff` показывает различия между файлами.

```sh
git diff
```

### 2.7. git show

Команда `git show` показывает содержимое коммита.

```sh
git show HEAD
```

### 2.8. git fetch

Команда `git fetch` получает изменения с удаленного сервера.

```sh
git fetch origin
```

### 2.9. git merge

Команда `git merge` объединяет изменения из одной ветки в другую.

```sh
git merge other-branch
```

### 2.10. git clean

Команда `git clean` удаляет все неиндексированные файлы в репозитории Git.

```sh
git clean -d -f
```

## 3. Управление ветками гита

### 3.1. git branch -d

Команда `git branch -d` удаляет определенную ветку.

```sh
git branch -d new-branch
```

### 3.2. git branch -m

Команда `git branch -m` переименовывает ветку.

```sh
git branch -m main-branch new-branch
```

### 3.3. git branch --list

Команда `git branch --list` показывает список всех веток.

```sh
git branch --list
```

### 3.4. git branch --delete

Команда `git branch --delete` удаляет определенную ветку.

```sh
git branch --delete old-branch
```

## 3. Управление удаленными репозиториями гита

### 3.1. git remote

Команда `git remote` показывает список удаленных репозиториев.

```sh
git remote
```

### 3.2. git remote add

Команда `git remote add` создает новый удаленный репозиторий.

```sh
git remote add origin https://github.com/username/repo.git
```

### 3.3. git remote rm

Команда `git remote rm` удаляет удаленный репозиторий.

```sh
git remote rm origin
```

## 4. Управление удаленными репозиториями гита

### 4.1. git remote rename

Команда `git remote rename` переименовывает удаленный репозиторий.

```sh
git remote rename origin old-name
```

### 4.2. git remote set-url

Команда `git remote set-url` изменяет URL удаленного репозитория.

```sh
git remote set-url origin https://github.com/username/repo.git
```

### 4.3. git remote show

Команда `git remote show` показывает информацию о удаленном репозитории.

```sh
git remote show origin
```

## 5. Команды гита для работы с пользователями и правами доступа

### 5.1. git config

Команда `git config` показывает и изменяет настройки Git.

```sh
git config --global user.name "John Doe"
```

### 5.2. git config --list

Команда `git config --list` показывает все настройки Git.

```sh
git config --list
```

### 5.3. git config --global

Команда `git config --global` изменяет настройки Git глобально.

```sh
git config --global user.name "John Doe"
```

### 5.4. git config --local

Команда `git config --local` изменяет настройки Git локально.

```sh
git config --local user.name "John Doe"
```

### 5.5. git config --system

Команда `git config --system` изменяет настройки Git системно.

```sh
git config --system user.name "John Doe"
```

### 5.6. git credential

Команда `git credential` управляет учетными данными для удаленного сервера.

```sh
git credential-cache
```

### 5.7. git credential-store

Команда `git credential-store` хранит учетные данные для удаленного сервера.

```sh
git credential-store /path/to/store
```

## 6. Команды гита для работы с тегами

### 6.1. git tag --delete

Команда `git tag --delete` удаляет тег.

```sh
git tag --delete v1.0
```

### 6.2. git tag --list
